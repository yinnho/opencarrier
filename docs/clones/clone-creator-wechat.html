用对话创造 AI 分身的分身

Clone Creator：OpenCarrier 的元分身

它是 OpenCarrier 里最特殊的分身：不服务用户，而是创造其他分身。

什么是 Clone Creator

Clone Creator 是 OpenCarrier 内置的元分身——一个专门帮用户设计和创建新 AI 分身的分身。

普通分身是"做事的"：客服分身回答问题、编程分身写代码、研究分身分析数据。

Clone Creator 不做这些——它只做一件事：帮你把脑海中的想法变成一个完整可用的 AI 分身。

它就像一个经验丰富的产品经理加技术顾问：你告诉它"我需要一个能自动回复电商售后的分身"，它会一步步引导你定义人格、设计技能、构建知识库，最终一键打包安装。

一个完整的创作团队

Clone Creator 不是单打独斗，它手下有 3 个专业子代理，按需派出协作：

Clone Creator（架构师 — 总指挥）
　│
　├── skill-designer（技能设计师）
　│　　专门设计分身的技能文件
　│
　├── agent-designer（代理设计师）
　│　　专门设计子代理文件
　│
　└── quality-reviewer（质量审查员）
　　　　打包前的最后一道关卡

这不是简单的任务拆分，而是真正的专业分工。skill-designer 只关心技能设计，不会越界写人格；quality-reviewer 只做检查，不修改任何文件。

八步创作流程

Clone Creator 的工作方式不是一口气生成所有东西，而是分步引导，像一个靠谱的产品经理那样"先理解需求，再动手"：

1 定位 — 问清楚分身要做什么。理解场景：给谁用？在什么平台工作？

2 人格 — 帮助定义分身的"灵魂"。生成 SOUL.md——性格、语气、边界。人格和行为严格分离。

3 知识 — 根据分身用途，规划知识文件：行业知识、FAQ、产品信息、流程指南。

4 技能（派出 skill-designer）— 精确的触发条件 + 精简的工具组合 + 清晰的执行步骤。

5 子代理（可选，派出 agent-designer）— 多角色协作时，设计每个子代理的独立指令和工具白名单。

6 系统指令 — 生成 system_prompt.md：角色定位、核心能力、工作流程、行为约束。

7 质量审查（派出 quality-reviewer）— 必需文件、YAML 格式、人格与行为分离——只有通过审查才进入打包。

8 打包安装 — 调用 clone_install 一次性完成所有文件打包、创建工作区、安装启动。

真实的创作过程

用户只需要对话：

👤 用户：我想创建一个电商客服分身

🤖 Clone Creator：好的！这个分身主要处理什么类型的咨询？售前咨询、售后问题、还是全都要？

👤 用户：主要是售后，退货换货这些

🤖 Clone Creator：明白了。给它起个名字吧？建议用英文短横线格式，比如 after-sales-support

👤 用户：就叫 after-sales 吧

🤖 Clone Creator：好的。它应该是什么性格？我建议：专业但不冷冰冰，像有经验的客服大姐，说话简洁，给方案不给废话

👤 用户：可以，就这样

🤖 Clone Creator：现在规划一下它需要什么知识...

... 几轮对话后 ...

✅ Clone Creator：分身创建完成！
名称：after-sales
质量评分：82/100
知识文件：3 个（退货政策、换货流程、常见问题）
技能：2 个（售后咨询处理、退款进度查询）
你现在可以直接和它对话了

从想法到可用的 AI 分身，一轮对话搞定。

不只是创建，还有进化

创建只是起点。安装后，分身进入 OpenCarrier 的生命周期管理：

• 自动学习：每次对话后自动提取新知识、发现知识缺口
• 知识过期：30 天未访问标记为 stale，60 天后删除
• 技能进化：同类请求 3 次以上自动生成新技能
• 质量评估：系统定期评估分身健康度

这些全部由 OpenCarrier 系统自动完成，分身本身无感知。

设计理念

OpenCarrier 大可以选择做一个 GUI 界面、一个表单向导来创建分身。但用分身来造分身有几个不可替代的优势：

1️⃣ 自然语言即配置
不需要学习任何配置格式。你描述"我想要一个温柔的客服"，Clone Creator 自然理解并转化为 SOUL.md 的人格描述。

2️⃣ 渐进式引导
表单需要一次填完所有字段。对话可以一步步来——先确定定位，再聊人格，再规划知识。用户可以随时补充、修改、推翻重来。

3️⃣ 专业分工
skill-designer 专注技能设计，quality-reviewer 专注质量检查。每个角色都是独立优化的，不会互相干扰。

4️⃣ 可进化
Clone Creator 自身也是一个分身。随着创建的分身越来越多，它会学习什么样的分身质量更高、什么模式更有效。

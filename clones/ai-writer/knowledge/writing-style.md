# 公众号 AI 科技写作风格指南

## 文章结构

### 标题
- 7 种标题公式：反常识、悬念、对比、痛点、数据冲击、实用技巧、工具推荐
- 标题不超过 30 字
- 必须包含关键词（AI、Agent、大模型等）

### 开头
- 前 3 句话必须给出核心信息或悬念
- 不用"近年来"、"随着...的发展"等套话开头
- 可以用：一个数据、一个反常识观点、一个场景问题

### 正文
- 每段不超过 3-4 句话
- 关键信息加粗
- 长文用小标题分节
- 技术概念第一次出现时给出简短解释

### 结尾
- 给出明确的行动指引或思考
- 不用"总而言之"、"综上所述"
- 可以用：一个问题、一个预测、一个建议

## 语言规范

### 术语处理
- 英文术语保留原文：Agent、LLM、RAG、MCP、Prompt
- 第一次出现时括号注释：(检索增强生成, RAG)
- 不生造中文翻译

### 反 AI 味清单
以下表达禁止使用：
- "值得注意的是"、"不可否认"、"毋庸置疑"
- "总而言之"、"综上所述"、"一言以蔽之"
- "在这个时代"、"在...的今天"
- "让我们一起"、"相信在未来"
- 连续使用"首先...其次...最后..."结构

### 替代方案
- 用具体数据替代"很多"、"大量"
- 用案例替代"例如...等"
- 用对比替代"非常好"、"非常强大"
- 段落间用内容逻辑自然过渡

## 文章类型模板

### 产品文章
```
[痛点场景] → [现有方案的局限] → [OpenCarrier/Aginx 的解决方式] → [实际效果/案例] → [上手方式]
```

### 行业分析
```
[现象/数据] → [背后的原因] → [技术解读] → [行业影响] → [我们的判断]
```

### 热点评论
```
[发生了什么] → [为什么重要] → [深度分析] → [对开发者的意义]
```

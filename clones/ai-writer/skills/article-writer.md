# Article Writer

从话题到完成 Markdown 文章的写作流程。

## Process

### 1. 确认话题和方向

- 明确文章类型（产品/行业/热点/教程）
- 确认目标读者（开发者/产品经理/普通读者）
- 确认字数目标

### 2. 素材研究

- 用 web_search 搜索相关话题的最新信息和数据
- 用 web_fetch 深入阅读关键来源
- 对于产品文章，查阅 knowledge/product-guide.md 中的素材
- 记录关键数据和引用来源

### 3. 列大纲

输出文章大纲，包含：
- 标题（2-3 个候选）
- 各节小标题
- 每节的核心论点（一句话）

等用户确认大纲后再写正文。

### 4. 撰写正文

按大纲逐节写作，遵循 knowledge/writing-style.md 中的风格指南：

- **开头**: 3 句话内给出核心信息或悬念
- **正文**: 每段 3-4 句，小标题分节，关键信息加粗
- **结尾**: 明确的行动指引或思考
- **反 AI 味**: 不用"值得注意的是"、"总而言之"等套话

### 5. 自审

写完后检查：
- [ ] 标题是否吸引人且准确
- [ ] 开头 3 句是否抓住读者
- [ ] 有无套话/AI 味表达
- [ ] 数据是否标注来源
- [ ] 字数是否在目标范围内
- [ ] 结尾是否有明确的收束

### 6. 输出

输出完整的 Markdown 文章。

## Important Principles

- 先研究再动笔，不凭印象写技术内容
- 数据必须通过搜索验证，不编造
- 引用要标注来源
- 用户可以要求修改任何部分，灵活调整
